# Web编译器/解释器实现

## 描述
在浏览器中实现一个编程语言的编译器或解释器，包括词法分析、语法分析、代码生成和执行环境。

## 要求
1. 使用JavaScript/TypeScript实现
2. 选择以下方案之一：
   - 实现一个现有简单语言的解释器（如Lisp、Scheme、Basic）
   - 设计并实现一个自定义语言
   - 创建一个特定领域语言(DSL)
3. 实现以下编译/解释阶段：
   - 词法分析（Lexer）
   - 语法分析（Parser）
   - 抽象语法树(AST)生成
   - 语义分析
   - 代码生成或解释执行
4. 创建一个Web IDE，包含：
   - 代码编辑器（语法高亮、自动完成）
   - 调试工具（断点、单步执行、变量查看）
   - 可视化执行过程
   - 错误报告和提示
5. 实现基本的标准库
6. 支持模块化和导入系统
7. 优化执行性能

## 输入
无特定输入，需要从头创建编译器/解释器项目。

## 期望输出
1. 完整的编译器/解释器源代码
2. Web IDE实现
3. 语言规范文档
4. 示例程序
5. 编译/解释过程说明
6. 性能分析报告

## 评估标准
1. 编译/解释过程的正确实现
2. 语言设计的合理性（如适用）
3. Web IDE的功能和用户体验
4. 错误处理和报告的质量
5. 执行性能和优化
6. 代码质量和可维护性
7. 文档的清晰度和完整性
8. 示例程序的功能展示
